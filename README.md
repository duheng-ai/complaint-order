# 通道投诉订单查询技能 (complaint-order)

自动解析投诉信息，查询订单并格式化输出（支持 3 种订单号格式解析，401 自动刷新 token）

## 🚀 功能特性

- ✅ **自动解析投诉信息** - 提取联系方式、投诉内容、订单号
- ✅ **支持 3 种订单号格式** - 火脸订单号、微信官方订单号、支付宝订单号
- ✅ **智能日期解析** - 从订单号提取日期，精准查询单日数据
- ✅ **401 自动刷新 token** - Token 失效时自动重新登录并更新全局 token
- ✅ **批量处理** - 支持一次处理多条投诉（空行分隔）
- ✅ **格式化输出** - 服务商、商户 ID、商户名称、通道投诉类型、订单详情

---

## 📋 订单号解析规则

| 规则 | 订单号前缀 | 示例 | 日期提取方式 | 显示名称 |
|------|-----------|------|-------------|----------|
| 规则 1 | `1026xxx` | `1026081200001329` | 第 4-6 位 = 年中第几天 | 火脸订单号 |
| 规则 2 | `42000xxxxx` | `4200003100202604027237553182` | 第 10-17 位 = YYYYMMDD | 官方订单号 |
| 规则 3 | `20xxxxxx` | `2026040223001419901446808099` | 前 8 位 = YYYYMMDD | 官方订单号 |

### 解析示例

```
订单号：1026081200001329
解析：2026 年第 081 天 → 2026-03-22
查询范围：2026-03-22 00:00:00 ~ 23:59:59

订单号：4200003100202604027237553182
解析：第 10-17 位 = 20260402 → 2026-04-02
查询范围：2026-04-02 00:00:00 ~ 23:59:59

订单号：2026040223001419901446808099
解析：前 8 位 = 20260402 → 2026-04-02
查询范围：2026-04-02 00:00:00 ~ 23:59:59
```

## 🔧 安装说明

### 前置条件

- OpenClaw 已安装并运行
- Node.js 14+ 已安装
- **火脸运营后台账号** - 用于 Token 自动刷新（需自行准备）

### 安装步骤

1. **确认技能位置**
   ```
   ~/.openclaw/skills/complaint-order/
   ```

2. **配置账号密码**
   
   编辑 `index.js`，修改顶部 **配置区域** 的参数：
   
   ```javascript
   // ========================
   // ⚙️ 配置区域（使用前请修改）
   // ========================
   const CONFIG = {
     // 火脸运营后台账号配置
     LOGIN: {
       phone: "您的手机号",      // ⚠️ 请修改为您的手机号
       password: "您的密码",     // ⚠️ 请修改为您的密码
       system: "operation",
       type: "password"
     },
     // API 配置（一般无需修改）
     API: {
       baseUrl: "https://api.lianok.com",
       loginEndpoint: "/common/v1/user/login",
       queryEndpoint: "/operation/v1/flow/selectByPage"
     },
     // 初始 Token（可选，会被自动刷新）
     initialToken: "ca83f4d3812f42688d6052bc4fba5d35"
   };
   ```
   
   > ⚠️ **重要**: 
   > - 必须修改 `phone` 和 `password` 为您自己的火脸运营后台账号
   > - 默认配置仅用于演示，不修改将导致查询失败
   > - 请妥善保管您的账号密码，不要提交到版本控制系统

3. **安装依赖**
   ```bash
   cd ~/.openclaw/skills/complaint-order
   npm install
   ```

4. **重启网关**（如需要）
   ```bash
   openclaw gateway restart
   ```

5. **验证配置**
   ```bash
   node run.js
   ```

## ⚙️ 配置参数说明

| 参数 | 说明 | 是否必须修改 |
|------|------|-------------|
| `LOGIN.phone` | 火脸运营后台手机号 | ✅ 必须 |
| `LOGIN.password` | 火脸运营后台密码 | ✅ 必须 |
| `API.baseUrl` | API 基础 URL | ❌ 一般无需修改 |
| `API.loginEndpoint` | 登录接口路径 | ❌ 一般无需修改 |
| `API.queryEndpoint` | 订单查询接口路径 | ❌ 一般无需修改 |
| `initialToken` | 初始 Token（可选） | ❌ 可选 |
