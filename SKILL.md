# complaint-order - 通道投诉订单查询

## 触发条件

当用户消息包含以下任意关键词时触发：
- `联系方式`
- `投诉内容`
- `订单号`

## 功能说明

- **自动解析投诉信息** - 提取联系方式、投诉内容、订单号
- **支持 3 种订单号格式** - 火脸订单号、微信官方订单号、支付宝订单号
- **智能日期解析** - 从订单号提取日期，精准查询单日数据
- **401 自动刷新 token** - Token 失效时自动重新登录
- **批量处理** - 支持一次处理多条投诉

## 订单号解析规则

| 规则 | 订单号前缀 | 示例 | 日期提取方式 | 显示名称 |
|------|-----------|------|-------------|----------|
| 规则 1 | `1026xxx` | `1026081200001329` | 第 4-6 位 = 年中第几天 | 火脸订单号 |
| 规则 2 | `42000xxxxx` | `4200003100202604027237553182` | 第 10-17 位 = YYYYMMDD | 官方订单号 |
| 规则 3 | `20xxxxxx` | `2026040223001419901446808099` | 前 8 位 = YYYYMMDD | 官方订单号 |

## 输入格式

```
用户投诉内容：充值 249 元，网卡的不行，看不了直播
用户联系方式：18876509647
订单号：4200003034202603317170467000
```

## 输出格式

```
所属服务商：杭州七矛科技有限公司
商户 ID：xxx 商户名称：xxx 存在 xxx 通道投诉
消费者联系方式：xxx
火脸订单号/官方订单号：xxx
订单金额：xxx
支付时间：xxx
投诉内容：xxx
```

## 查询范围

| 类型 | 时间范围 |
|------|----------|
| 订单查询 | 订单号解析出的具体日期（单日） |
| 查询策略 | 先查 orderNo，无结果再查 topChannelOrderNo |

## 特性

- ✅ **401 自动刷新 token** - Token 失效时自动重新登录并更新全局 token
- ✅ **精准日期解析** - 从订单号提取日期，避免大范围查询
- ✅ **3 种订单号格式支持** - 覆盖火脸、微信、支付宝订单
- ✅ **批量处理** - 支持一次处理多条投诉（空行分隔）

## API

- **接口**: `https://api.lianok.com/operation/v1/flow/selectByPage`
- **登录**: `https://api.lianok.com/common/v1/user/login`
- **Header**: `accessToken`, `client: WEB`

> ⚠️ **注意**: Token 自动刷新功能需要配置您自己的火脸运营后台账号密码（手机号 + 密码），请修改 `index.js` 中的 `getNewToken()` 函数。
